# 02. 고정지출 등록 (Add Fixed Expense)

## 1. 화면 개요
사용자가 매월 발생하는 고정지출 항목을 등록하거나 수정하는 화면입니다.
지출 항목명, 금액, 결제일, 카테고리, 결제 수단 등의 필수 정보를 입력받아 관리합니다.

## 2. 주요 기능

### 2.1. 정보 입력 (Data Entry)
-   **지출 항목명 (Name)**: 무엇에 대한 지출인지 명확히 입력합니다. (예: 월세, 넷플릭스, 관리비 등)
-   **금액 (Amount)**: 매월 지출되는 금액을 입력합니다.
-   **결제일 (Payment Day)**: 매월 며칠에 결제되는지 선택합니다. (1일 ~ 말일)
-   **카테고리 (Category)**: 지출 성격에 맞는 카테고리를 선택합니다.
-   **결제 수단 (Payment Method)**: 어떤 수단(카드, 계좌 등)으로 결제되는지 선택합니다.
-   **메모 (Memo)**: 추가적인 설명이 필요할 경우 입력합니다. (선택 사항)

### 2.2. 수정 및 삭제 (Edit & Delete)
-   등록된 항목을 수정 모드로 진입 시 기존 정보가 입력 폼에 미리 채워집니다.
-   수정 모드에서는 **삭제 버튼**이 활성화되어 해당 고정지출 항목을 삭제할 수 있습니다.

## 3. UI 구성 요소 (Component Layout)

### A. 헤더 (Header)
-   좌측: 뒤로가기 버튼 (`<`)
-   중앙: 화면 타이틀 ("고정지출 등록" 또는 "고정지출 수정")
-   우측: 저장(완료) 버튼 (유효성 검사 통과 시 활성화)

### B. 입력 폼 영역 (Form Section)
-   **Category Selector**:
    -   아이콘 그리드 형태 또는 드롭다운으로 제공
    -   선택 시 시각적 피드백 (Highlight) 제공
-   **Amount Input**:
    -   숫자 키패드 또는 숫자 입력 필드
    -   천 단위 콤마(,) 자동 포맷팅
    -   원(₩) 단위 표시
-   **Details Input**:
    -   Name: 텍스트 입력 필드
    -   Day: 1~31일 선택 (Select/Spinner)
    -   Method: 등록된 결제 수단 목록에서 선택 (Select/Modal)
    -   Memo: 텍스트 에어리어 (Textarea) 또는 한 줄 입력

### C. 하단 버튼 영역 (Footer Actions) - *Edit Mode Only*
-   **삭제 버튼**: 붉은색 텍스트 또는 버튼으로 위험 요소임을 표시
-   클릭 시 재확인 모달(Confirm Modal) 노출

## 4. 데이터 요구사항 (Data Requirements)

### API: `POST /api/expenses` (신규 등록)
-   **Request Body**:
    ```json
    {
      "name": "넷플릭스 구독",
      "amount": 17000,
      "paymentDay": 15,
      "categoryId": "cat_001",
      "paymentMethodId": "pm_001",
      "memo": "프리미엄 요금제"
    }
    ```
-   **Response**: `201 Created` / `{ "id": "exp_12345", ... }`

### API: `PUT /api/expenses/{id}` (수정)
-   **Path Parameter**: `id` (Expense ID)
-   **Request Body**: 위와 동일 (수정된 내용 포함)
-   **Response**: `200 OK` / `{ "id": "exp_12345", ... }`

### API: `DELETE /api/expenses/{id}` (삭제)
-   **Path Parameter**: `id` (Expense ID)
-   **Response**: `204 No Content`

## 5. 인터랙션 (Interactions)
1.  **필수 값 검증**: 항목명, 금액, 결제일 등 필수 값이 비어있을 경우 저장 버튼이 비활성화되거나 클릭 시 경고 메시지를 표시합니다.
2.  **금액 입력**: 금액 입력 시 실시간으로 천 단위 구분 기호가 적용됩니다.
3.  **저장 완료**: 저장 성공 시 알림(Toast)을 띄우고 이전 화면(목록 또는 대시보드)으로 자동 이동합니다.
4.  **삭제 확인**: 삭제 버튼 클릭 시 "정말로 삭제하시겠습니까?" 묻는 팝업을 띄워 실수를 방지합니다.
