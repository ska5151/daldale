# 고정지출 관리 API 문서

## 1. 개요
고정지출 항목을 등록, 수정, 삭제하는 API 명세서입니다.

## 2. API 목록

### 2.1 고정지출 등록
- **URL**: `/api/expenses`
- **Method**: `POST`
- **Description**: 새로운 고정지출 항목을 생성합니다.

#### Request Body
| Field | Type | Required | Description |
|---|---|---|---|
| name | string | Y | 지출 항목명 |
| amount | number | Y | 금액 |
| paymentDay | integer | Y | 결제일 (1~31) |
| categoryId | integer | Y | 카테고리 ID (`categories` table PK) |
| paymentMethodId | integer | Y | 결제 수단 ID (`payment_methods` table PK) |
| memo | string | N | 메모 |

#### Response
- **Success (201 Created)**
```json
{
  "code": 201,
  "message": "등록 성공",
  "data": {
    "expenseId": 123
  }
}
```

### 2.2 고정지출 수정
- **URL**: `/api/expenses/{expenseId}`
- **Method**: `PUT`
- **Description**: 기존 고정지출 항목을 수정합니다.

#### Path Parameters
| Parameter | Type | Required | Description |
|---|---|---|---|
| expenseId | integer | Y | 수정할 지출 항목 ID |

#### Request Body
| Field | Type | Required | Description |
|---|---|---|---|
| name | string | Y | 지출 항목명 |
| amount | number | Y | 금액 |
| paymentDay | integer | Y | 결제일 (1~31) |
| categoryId | integer | Y | 카테고리 ID |
| paymentMethodId | integer | Y | 결제 수단 ID |
| memo | string | N | 메모 |

#### Response
- **Success (200 OK)**
```json
{
  "code": 200,
  "message": "수정 성공",
  "data": {
    "expenseId": 123
  }
}
```

### 2.3 고정지출 삭제
- **URL**: `/api/expenses/{expenseId}`
- **Method**: `DELETE`
- **Description**: 고정지출 항목을 삭제합니다.

#### Path Parameters
| Parameter | Type | Required | Description |
|---|---|---|---|
| expenseId | integer | Y | 삭제할 지출 항목 ID |

#### Response
- **Success (204 No Content)**
(Body 없음)

### 2.4 고정지출 상세 조회
- **URL**: `/api/expenses/{expenseId}`
- **Method**: `GET`
- **Description**: 특정 고정지출 항목의 상세 정보를 조회합니다. (수정 화면 진입 시 사용)

#### Response
- **Success (200 OK)**
```json
{
  "code": 200,
  "message": "조회 성공",
  "data": {
      "expenseId": 123,
      "name": "넷플릭스 구독",
      "amount": 17000,
      "paymentDay": 15,
      "categoryId": 1,
      "paymentMethodId": 2,
      "memo": "프리미엄 요금제"
  }
}
```
